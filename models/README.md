# Models

This folder contains trained machine learning model files.

## risk_classifier_pipeline.joblib

**Model:** TF-IDF + Logistic Regression pipeline
**Purpose:** Classifies user reports into Trust & Safety risk categories
**Trained on:** data/synthetic_user_reports.csv (500 synthetic records)
**Target variable:** risk_category
**Accuracy:** 85% on test set (100 samples)

This model file is generated by running:

```bash
python notebooks/03_risk_classifier.py
```

The script trains a scikit-learn pipeline combining:

- TfidfVectorizer (max_features=5000, ngram_range=(1, 2))
- LogisticRegression (solver=lbfgs, max_iter=1000, random_state=42)

### Loading the model

```python
import joblib

pipeline = joblib.load("models/risk_classifier_pipeline.joblib")

# Predict risk category from a user report
report = ["Someone has been sending me threatening messages repeatedly"]
prediction = pipeline.predict(report)
print(prediction)
# Output: ['Harassment']
```

### Why is the .joblib file not committed?

Binary model files (like .joblib) are not committed to this repository because:

1. They can be fully reproduced by running `python notebooks/03_risk_classifier.py`.
2. Binary files increase repository size without adding readability.
3. Recruiters and reviewers can regenerate the model in under 30 seconds after installing requirements.

To regenerate:

```bash
pip install -r requirements.txt
python src/generate_synthetic_tickets.py
python notebooks/03_risk_classifier.py
```

This will create `models/risk_classifier_pipeline.joblib` locally.
